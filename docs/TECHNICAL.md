# AI Memory MCP ‚Äî Technical Documentation

## Stack (Docker Compose)

| Component | Source | Notes |
|-----------|--------|-------|
| Python | `mcp/Dockerfile` ‚Äî `python:3.12-slim` | MCP service |
| FastMCP | `mcp/requirements.txt` | MCP over SSE (`mcp/main.py`) |
| PostgreSQL | `pgvector/pgvector:pg16` | DB `ai_memory`, extension `vector`, `pg_trgm` |
| Neo4j | `neo4j:5-community` | APOC; only on internal Docker network |
| Gitea | `gitea/gitea:1.23.4` | HTTP `127.0.0.1:3000` on host + SSH **2222** public |
| Caddy | `caddy:2-alpine` | **`network_mode: host`** ‚Äî TLS on host `:80`/`:443`; sees real client IP for allowlist (Linux only) |
| Admin UI | `admin/` (FastAPI) | Token/user management |

There is **no** host install of PostgreSQL, Gitea, or Caddy ‚Äî everything runs in containers. Versions of Python libraries are pinned only in `mcp/requirements.txt` (no separate lockfile in repo).

## Network

- **`backend`**: `internal: true` ‚Äî Postgres, Neo4j, Gitea, MCP, Admin; **no** inbound from the internet on this network. DBs are not published on the host.
- **`egress`**: non-internal network attached to **Gitea** and **MCP** so they keep outbound internet (webhooks, `git clone` from upstream, LLM/embeddings APIs). Replaces the old separate `proxy` bridge.
- **Caddy** uses **`network_mode: host`** (Linux): listens on the host‚Äôs `80`/`443`/`443` UDP, so **`remote_ip` in the Caddyfile matches the real client** (IP allowlist works). Upstreams are **`127.0.0.1:3000` / `:8000` / `:8080`** ‚Äî HTTP for Gitea/MCP/Admin is published **only on localhost** on the host, not on the LAN.

**Docker Desktop (Windows/macOS):** `network_mode: host` is **not** supported like on Linux; deploy Caddy on a **Linux** host or use another edge proxy that sets `X-Forwarded-For` and configure Caddy `trusted_proxies` (not included in this repo).

Databases are **not** mapped to host ports. Access from the host to Postgres/Neo4j is only via `docker compose exec`, not `localhost:5432`.

## Repository layout (deploy root)

```
./
  docker-compose.yml    # stack definition
  setup.sh              # guided .env + first deploy
  .env.example          # variable template
  caddy/
    Caddyfile
    entrypoint.sh       # whitelist ‚Üí CADDY_ALLOWED_IPS
  mcp/                    # MCP image context
    main.py
    requirements.txt
    config.py
    tools/               # knowledge, graph, semantic, gitea_tools, git_tools
  admin/                  # Admin UI image context
  init-db/                # Postgres init (schema + first user)
```

Runtime config: **`.env`** in project root (generated by `setup.sh` or copied from `.env.example`). Not: `/etc/gitea`, `/etc/caddy` on the host ‚Äî those paths apply only **inside** images if at all; this project mounts `./caddy/*` into the Caddy container.

## Database

- **DB name:** `ai_memory`
- **App DB user:** `mcpuser` (password `MCP_DB_PASSWORD` in `.env`)
- **Superuser** for dumps: `postgres` / `POSTGRES_PASSWORD`

### `categories` (excerpt)

`user_id` is required (multi-tenant). `path` is unique globally in schema (see `init-db/init.sh` for full DDL).

### Managing the stack

```bash
cd /path/to/ai-memory-mcp
docker compose ps
docker compose logs -f mcp
docker compose exec postgres psql -U mcpuser -d ai_memory
docker compose up -d --build mcp
docker compose down
```

## Caddy

- Config: `caddy/Caddyfile` ‚Äî `reverse_proxy` targets **`127.0.0.1`** (see `docker-compose.yml` port mappings), not Docker DNS names.
- Optional IP allowlist (Gitea + Admin hostnames only): `CADDY_WHITELIST_*` in `.env` (`caddy/entrypoint.sh`). MCP hostname is not filtered by IP.

## Category tree

`init-db/init.sh` creates schema and the first admin user only. The **example** tree in [README.md](README.md) is a recommended layout ‚Äî create categories with `create_category` / Admin as needed.
